## 計算邏輯

### 1. 同時使用量計算

由於碗的周轉時間為 1 小時（使用 + 清洗），在任一時段同時佔用的碗數為：

```
同時使用量 = 當前時段需求 + 上一時段需求（還在清洗中）
```

例如 12:00 的麵碗同時使用量：
- 12:00 新需求：1.84 碗
- 11:00 的碗還在清洗：0.79 碗
- 同時使用：1.84 + 0.79 = 2.63 碗

### 2. 安全庫存計算

```
保守方案 = 尖峰同時使用量 × 1.5 + 1（備用）
標準方案 = 尖峰同時使用量 × 2.0 + 2（備用）
充裕方案 = 尖峰同時使用量 × 2.5 + 2（備用）
```

## 風險評估

### 低風險情境
- 碗數足夠應對日常營運
- 即使週六尖峰也不會缺碗

### 潛在風險
1. **特殊節日**：如遇農曆新年、國定假日，客流可能增加 50-100%
2. **破損損耗**：建議每季檢查碗數，補充損耗
3. **清洗延遲**：如清洗時間超過 1 小時，需額外準備

### 建議

| 情境 | 建議方案 | 碗數 |
|------|---------|------|
| 保守經營 | 保守方案 | 麵 5 + 飯 3 = 8 個 |
