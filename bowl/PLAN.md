# 碗需求量計算系統 - 多代理架構設計

## 問題定義

**目標**：計算從免洗餐具轉換為可重複使用碗盤時，需要購買多少個碗才不會「開天窗」

**約束條件**：
- 碗周轉時間：1 小時（清洗時間）
- 內用比例：50%
- 外帶比例：50%（使用免洗餐具，不需碗）
- **碗分兩種**：
  - 🍜 **麵碗**：用於麵 + Combo(含麵)
  - 🍚 **飯碗**：用於飯 + Combo(含飯)
- **特殊高峰**：週六/假日中午 12:00-14:00

## 計算公式

### 1. 每小時碗需求（分平日/週六）

```
# 平日需求
weekday_demand[hour] = Σ(qty on Mon,Tue,Fri) / weekday_count × 0.5（內用）

# 週六需求（高峰基準）
saturday_demand[hour] = Σ(qty on Sat) / saturday_count × 0.5（內用）
```

### 2. 同時使用碗數（考慮 1hr 周轉）

```
concurrent_bowls[hour] = demand[hour] + demand[hour-1]

例如 12:00 時：
• 12:00 新需求的碗正在使用
• 11:00 用完的碗還在清洗中
• 兩者都佔用碗，需同時準備
```

### 3. 週六高峰最大使用量

```
# 以週六 12:00-14:00 為最高峰
peak_noodle_bowls = max(noodle_concurrent[12], noodle_concurrent[13], noodle_concurrent[14])
peak_rice_bowls = max(rice_concurrent[12], rice_concurrent[13], rice_concurrent[14])
```

### 4. 建議購買量（雙碗分開）

```
🍜 麵碗：
  保守 = peak_noodle × 1.5 + 1（備用）
  標準 = peak_noodle × 2.0 + 2（備用）
  充裕 = peak_noodle × 2.5 + 2（備用）

🍚 飯碗：
  保守 = peak_rice × 1.5 + 1（備用）
  標準 = peak_rice × 2.0 + 2（備用）
  充裕 = peak_rice × 2.5 + 2（備用）
```

## 執行過程中發現的問題與解決方案

### 問題 1：時區轉換錯誤 🔴

**問題描述**：
原始計劃假設 CSV 數據中的 Date 欄位為 UTC 時區，需轉換為 America/New_York。

**發現過程**：
1. 初次計算結果顯示所有交易集中在 19:00-20:00，不合理
2. 檢查 CSV 欄位發現有獨立的 `Time` 欄位和 `Time Zone` 欄位
3. `Time Zone` 欄位顯示為 `Taipei`（UTC+8）

**錯誤的做法**：
```python
# ❌ 錯誤：假設數據為 UTC
df['DateTime'] = pd.to_datetime(df['Date'])
df['DateTime'] = df['DateTime'].dt.tz_localize('UTC').dt.tz_convert(ny_tz)
```

**正確的解決方案**：
```python
# ✅ 正確：合併 Date + Time，從 Taipei 轉換為 New York
taipei_tz = pytz.timezone('Asia/Taipei')
ny_tz = pytz.timezone('America/New_York')

df['DateTime'] = pd.to_datetime(df['Date'] + ' ' + df['Time'])
df['DateTime'] = df['DateTime'].dt.tz_localize(taipei_tz).dt.tz_convert(ny_tz)
```

**影響**：
- 時區修正後，交易時間分布正確（11:00-18:00 為主）
- 尖峰時段正確識別為 12:00

---

### 問題 2：營業日判定方式 🔴

**問題描述**：
原始計劃假設固定營業日為「週一、二、五、六」，但實際營業日可能不同。

**發現過程**：
1. 用戶反饋「有些天數沒有營業」
2. 檢查實際交易數據發現：
   - 原假設的週三、週四、週日也有交易記錄
   - 但時區修正後，這些變成 0 天

**錯誤的做法**：
```python
# ❌ 錯誤：使用固定假設
operating_days = [0, 1, 4, 5]  # Mon, Tue, Fri, Sat
df = df[df['DayOfWeek'].isin(operating_days)]
saturday_count = 29  # 假設值
```

**正確的解決方案**：
```python
# ✅ 正確：使用「有賣麵飯 = 營業」作為判定標準
bowl_df = df[df['BowlType'].notna()]  # 只看需要碗的品項
operating_dates = bowl_df.drop_duplicates('DateOnly')

# 統計每個星期幾的實際營業天數
saturday_count = len(operating_dates[operating_dates['DayOfWeek'] == 5])
```

**影響**：

| 項目 | 原假設 | 實際數據 |
|------|--------|----------|
| 週六天數 | 29 天 | 19 天 |
| 週日天數 | 0 天 | 0 天 |
| 週三天數 | 0 天 | 0 天 |

### 問題 3：Combo 麵品項識別 🟡

**問題描述**：
初次計算時 `combo_noodle` 總數為 0，但實際上應該有數據。

**發現過程**：
Data Auditor 警告：「⚠️ combo_noodle 總數為 0」

**原因**：
品項名稱中有拼寫錯誤和空格問題：
- `Combo--- noodles + Bubble Tea`（有空格）
- `Combo---Noodels + Floral Taiwan Tea`（Noodels 拼錯）

**錯誤的做法**：
```python
# ❌ 錯誤：只檢查 'noodles'
if 'combo' in item and 'noodles' in item:
    return 'combo_noodle'
```

**正確的解決方案**：
```python
# ✅ 正確：同時檢查 'noodle' 和 'noodels'
if 'combo' in item and ('noodle' in item or 'noodels' in item):
    return 'combo_noodle'
```

### 問題 4：CSV Date 欄位沒有時間 🟡

**問題描述**：
初次載入時假設 Date 欄位包含完整時間戳記。

**發現過程**：
```python
df['Date'].head()
# 輸出：2025-11-15, 2025-11-15, ...（只有日期）
```

**原因**：
Square 導出的 CSV 將日期和時間分成兩個欄位：
- `Date`: 日期（YYYY-MM-DD）
- `Time`: 時間（HH:MM:SS）

**解決方案**：
合併兩個欄位再進行時區轉換。
