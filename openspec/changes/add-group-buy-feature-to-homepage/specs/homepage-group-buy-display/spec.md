## Capability: Homepage Group Buy Display

### Description
此功能旨在 `taiwanway` 網站的首頁上，以清晰且吸引人的方式展示當前活躍中的團購活動。用戶無需導航至專門的團購頁面，即可快速瀏覽主要團購資訊，提高團購參與度。

### Scenarios

#### Scenario: Display Active Group Buy Listings on Homepage
**Given** 使用者訪問 `taiwanway` 網站的首頁
**And** 存在至少一個目前活躍且可供參與的團購活動
**When** 首頁加載完成
**Then** 首頁上應顯示一個「團購專區」或類似的區塊
**And** 該區塊應包含活躍團購活動的列表或卡片
**And** 每個團購活動應至少顯示以下資訊：
    *   團購名稱
    *   主視覺圖片
    *   簡短描述
    *   當前進度或參與人數
    *   截止日期
    *   一個點擊後可導向該團購詳情頁面的連結/按鈕

#### Scenario: No Active Group Buy Listings
**Given** 使用者訪問 `taiwanway` 網站的首頁
**And** 目前沒有任何活躍且可供參與的團購活動
**When** 首頁加載完成
**Then** 首頁上可以不顯示「團購專區」區塊
**Or** 首頁上顯示一個「目前沒有活躍團購」的提示訊息，並可引導至過往團購或未來預告頁面

### Inputs
*   `homepage`: 網站首頁的渲染請求。
*   `groupBuyData`: 一個包含活躍團購活動詳細資訊的數據集合（可能來自 API 或前端狀態）。

### Outputs
*   渲染至首頁的 HTML/DOM 元素，包含團購活動的視覺化展示。
*   對每個團購活動詳情頁的導航連結。

### Error Handling
*   **數據加載失敗**：如果團購數據加載失敗，首頁的團購區塊應 graceful 地處理，例如顯示「團購資訊加載失敗」或隱藏該區塊。
*   **數據格式錯誤**：如果接收到的團購數據格式不符合預期，應有容錯機制，避免首頁渲染崩潰。

### Non-Functional Requirements
*   **性能**：團購區塊的渲染不應顯著增加首頁的加載時間。數據獲取應高效。
*   **響應式設計**：在不同螢幕尺寸（桌面、平板、手機）下，團購區塊應能良好顯示。
*   **可訪問性**：團購區塊的內容應符合無障礙設計規範。
